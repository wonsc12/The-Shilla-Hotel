클래스명 규칙

  1. 소문자만 작성가능
  2. 단어와 단어 사이에는 -로구분 ex - tit-area

script 규칙

  1. 변수명/함수명 명사로 시작
  2. 낙타기법 사용 ex - currentIdx

이미지명 규칙

  1. 소문자만 작성가능
  2. 스네이크기법 사용 ex - bg_hotel_intro
  3. 최대 3단계 까지만 사용 ex - bg_hotel_intro
  4. 백그라운드 : bg_
    버튼 : btn_
    아이콘 : ico_
    이미지 : img_

주석 규칙

  1. 기본 작업 주석
    html
    <!-- 콘텐츠명 (s) -->
    <!-- 콘텐츠명 (e) --> css
    /* 콘텐츠명 */
    script
    //콘텐츠명
  2. 수정 작업 주석
    html
    한줄일 때 (html 마크업 뒤에 주석 달기)
    <!-- 수정 240317 박소현 : 제목변경 -->
    두줄이상일 때
    <!-- 수정 240317 박소현 : 레이아웃변경 (s) -->
    <!-- 수정 240317 박소현 : 레이아웃변경 (e) --> css
    css {} 괄호 뒤에 주석 달기
    /* 수정 240317 박소현: 컬러변경 */
    script
    위와 동일

css 작성 규칙

  1. 속성과 속성 사이에는 한칸 띄어쓰기, 세미콜론 뒤에는 한칸 띄어쓰기
    ex - .bg {display: block; width: 50px;}
  2. 다중선택자가 있을 경우 한칸씩 띄어쓰기
    ex -
    header,
    footer {background-color:#ccc;}
  3. css 속성 순서 (네이버 규칙 참고 후 적용)

깃허브 커밋메시지 규칙

  1. [feat] 박소현/js/클릭이벤트추가
  2. feat : 새로운 기능 추가, 기존의 기능을 요구 사항에 맞추어 수정
  3. fix : 기능에 대한 버그 수정
  4. build : 빌드 관련 수정
  5. chore : 패키지 매니저 수정, 그 외 기타 수정 ex - .gitignore
  6. ci : CI 관련 설정 수정
  7. docs : 문서(주석) 수정
  8. style : 코드 스타일, 포맷팅에 대한 수정
  9. refactor : 기능의 변화가 아닌 코드 리팩터링 ex) 변수 이름 변경
  10. test : 테스트 코드 추가/수정
  11. release : 버전 릴리즈
